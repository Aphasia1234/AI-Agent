# RAG

- LLM的知识来源于哪里？
   训练的时候会给它数据集。
- LLM 的幻觉？
   AIGC 生成的内容，是根据数据集训练的，所以会有一些幻觉。
   问了LLM不知道的问题，它会胡乱回答。
- RAG  可以解决LLM的幻觉。
  - LLM thinking planing
  - 检索增强
  - Augument Prompt
  - 如果没有匹配到 -> 回答“我不知道”

## RAG

- Retriver
   - 原始的prompt embedding 
   - 知识库 提前embedding好的
   - cosine 相似度计算

- 知识库
   - 专家知识库
   - 企业私有/安全知识库
   - 各种类型的文件， txt pdf mp3 video
   - 大的文件切片，document 文档碎片
   - embedding 化

- Augument 增强
   - 原始的Prompt 增加 检索出来的几段相关文档

- Generate 生成
   - LLM 拿到增强的Prompt 完美回答

## 向量表达？
   
- 关键词的文本匹配不能实现语义搜索
   - 查询文中提到的水果，prompt 包含水果的关键词，但是不能实现语义搜索。
- 向量 Vector 
   用数字表达一个存储的信息

   - 食用性 0：不能食用 1：可以食用
   - 硬度 0：液体 1：固体
     水果 [0.9,0.3] 食用性极高，硬度偏低
     苹果 [0.9,0.5] 食用性极高，硬度中等
     香蕉 [0.9,0.1] 食用性极高，硬度偏低
     石头 [0.1,0.9] 食用性偏低，硬度极高
    
- 语义搜索的流程
   - 向量每个维度有独特的语义(食用性，硬度)
   - 可视化 空间
   - cosine 相似度计算
